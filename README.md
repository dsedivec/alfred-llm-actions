# LLM Workflow for Alfred 5

A keyboard-driven Alfred workflow for sending prompts and selected text to LLMs without leaving your current app. Supports OpenAI, Anthropic, Google Gemini, and OpenRouter.

## Installation

1. Zip the contents of this folder into a file named `LLM Workflow.alfredworkflow`
2. Double-click the `.alfredworkflow` file to install it in Alfred
3. Open the workflow's settings in Alfred and set at least one API key (see below)

## Setup

### API Keys

Open Alfred Preferences → Workflows → LLM Workflow → Configure (the `[x]` button). Set the keys for the providers you want to use:

| Variable             | Provider    |
|----------------------|-------------|
| `OPENAI_API_KEY`     | OpenAI      |
| `ANTHROPIC_API_KEY`  | Anthropic   |
| `GEMINI_API_KEY`     | Google Gemini |
| `OPENROUTER_API_KEY` | OpenRouter  |

You only need keys for the providers whose models you plan to use.

### Select a Model

Type `llm-model` in Alfred to see all available models. Select one to make it the active model. Your choice persists across invocations until you change it.

Models are loaded from `models_default.yaml` (ships with the workflow) and optionally merged with `models.yaml` (user overrides). See [Model Configuration](#model-configuration) below.

## Usage

### Freeform Q&A

Type in Alfred:
```
ask What is the capital of France?
```
The response is copied to your clipboard and a notification confirms it.

### Continue a Conversation

```
ask-more Can you tell me more about that?
```
This appends to the previous exchange so the model has context from your last question.

### Manage Templates

Type `llm-templates` in Alfred to:

- **Edit** an existing template — select it from the list to open it in your default text editor
- **Create** a new template — type a name (e.g. `llm-templates code review`) and select "Create New Template" to generate a skeleton `.txt` file and open it for editing
- **Open the templates folder** — select "Open Templates Folder" to open it in Finder

### Text Transformation (Universal Actions)

1. **Select text** in any app
2. Press **⌘⇧\\** (or your configured Universal Action hotkey) to open Alfred's Universal Actions
3. Choose **"Send to LLM"**
4. A filterable list of templates appears — type to narrow it down (e.g. "sum" for Summarize)
5. Pick a template and wait for the response

Depending on the template's output mode, the result will be copied to your clipboard or pasted over your selection.

### Built-in Templates

| Template                  | Output    | Description                              |
|---------------------------|-----------|------------------------------------------|
| Summarize                 | clipboard | Summarize the selected text              |
| Rewrite / Improve Writing | paste     | Improve clarity and readability          |
| Fix Grammar & Spelling    | paste     | Fix errors without changing style        |
| Change Tone               | clipboard | Rewrite in a professional tone           |
| Translate to English      | clipboard | Translate text to English                |
| Break into Bullet List    | clipboard | Restructure into bullets without rewriting |

## Model Configuration

Models are defined in YAML files in the workflow directory:

- **`models_default.yaml`** — Ships with the workflow. Contains curated defaults for OpenAI, Anthropic, Gemini, and OpenRouter models. Do not edit this file (it will be overwritten on updates).
- **`models.yaml`** — Your overrides. Create this file to add custom models or remove defaults.

### Model Entry Format

```yaml
models:
  - label: My Custom Model        # Display name (unique identifier)
    provider: openai               # openai, anthropic, gemini, or openrouter
    model: ft:gpt-4o:my-org:id    # Model ID sent to the API
    params:                        # Optional: merged into API request
      temperature: 0.2
```

| Field      | Required | Description |
|------------|----------|-------------|
| `label`    | Yes      | Display name in Alfred. Must be unique across all models. |
| `provider` | Yes      | One of: `openai`, `anthropic`, `gemini`, `openrouter` |
| `model`    | Yes      | The model ID sent to the provider API |
| `params`   | No       | Dict deep-merged into the API request body |

### User Overrides (`models.yaml`)

Create `models.yaml` in the workflow directory to add or remove models:

```yaml
models:
  # Add a custom model
  - label: My Fine-tune
    provider: openai
    model: ft:gpt-4o:my-org:custom:id
    params:
      temperature: 0.2

  # Remove a specific default model by label
  - remove_defaults: "GPT-4o Mini"

  # Remove all GPT models
  - remove_defaults: "GPT-*"

  # Remove all OpenRouter models
  - remove_defaults: "*via OpenRouter*"

  # Remove ALL defaults (start from scratch)
  - remove_defaults: "*"
```

User-added entries are appended to the defaults. Use `remove_defaults:` to drop default models by label pattern — supports glob wildcards (`*`, `?`, `[seq]`, `[!seq]`). Removals only affect models from `models_default.yaml`, never your own entries in the same file.

### Params & Unified Shorthands

The `params` dict is deep-merged into the provider-specific request body. Two unified shorthands are translated automatically:

#### `reasoning`

Set to `auto`, `high`, `medium`, or `low`. Translated per provider:

- **Anthropic:** `thinking: {type: "enabled", budget_tokens: ...}` (`auto` uses `budget_tokens: 0`)
- **OpenAI:** `reasoning_effort: "high|medium|low"`
- **Gemini:** `thinking_config: {thinking_budget: ...}`
- **OpenRouter:** `reasoning_effort`

#### `web_search: true`

- **OpenAI:** `web_search_options: {search_context_size: "medium"}`
- **Anthropic:** `tools: [{type: "web_search_20250305"}]`
- **Gemini:** `tools: [{google_search: {}}]`
- **OpenRouter:** `plugins: ["web"]`

Raw provider-native params always override shorthands. Any other keys in `params` are passed through as-is.

## Custom Templates

Add your own templates by dropping a `.txt` file in the `templates/` folder. They appear automatically in the template list.

### Template Format

```
---
name: My Custom Template
output: clipboard
system: You are a helpful assistant.
---
Do something with the following text:

{{input}}
```

**Frontmatter fields:**

| Field    | Required | Description |
|----------|----------|-------------|
| `name`   | No       | Display name in Alfred (defaults to filename) |
| `output` | No       | `clipboard` (default), `paste`, or `largetype` |
| `system` | No       | System prompt for this template (overrides global) |

The `{{input}}` placeholder is replaced with the selected text or typed input.

### Output Modes

- **clipboard** — Response is copied to the clipboard. A notification confirms it.
- **paste** — Response is copied to the clipboard and immediately pasted, replacing the selected text.
- **largetype** — Response is copied to the clipboard and shown in a notification.

## Global System Prompt

You can set a default system prompt that applies to all invocations (unless a template specifies its own):

- **Via Alfred:** Set the `System Prompt` field in the workflow's configuration
- **Via file:** Edit `system_prompt.txt` in the workflow directory

The Alfred environment variable takes priority over the file.

## Error Handling

If something goes wrong (bad API key, rate limit, network error), a macOS notification will appear with the error message. There is no automatic fallback to another provider.

## File Structure

```
├── info.plist             # Alfred workflow definition
├── llm.py                 # Main script (API calls, templates, state, output)
├── select_model.py        # Model selection script filter
├── models_default.yaml    # Default model list (ships with workflow)
├── models.yaml            # User model overrides (create this yourself)
├── system_prompt.txt      # Global system prompt
├── templates/             # Prompt templates (built-in + custom)
│   ├── summarize.txt
│   ├── rewrite.txt
│   ├── fix_grammar.txt
│   ├── change_tone.txt
│   ├── translate.txt
│   └── bullet_list.txt
└── state/                 # Runtime state (active model, last conversation)
```
